HTML

  • HTML (HyperText Markup Language) is the language of the Web
  • Allows authors to publish information on the Web and, importantly, to link to other Web documents
  • Is a simple markup language

Markup

  • Sections of text are marked up with special tags that indicate what that section of text is
  • For example, there are special tags to indicate that a bit of text is a paragraph or a heading or a hyperlink
  • Tags act as instructions to the Web browser because once the browser knows what the text is it can decide how to display it
  • For example, a heading tag will tell the browser that that bit of text is a heading and so should be displayed larger than the rest of the text
  • Basically, HTML tags define the structure of a Web page

HTML Tags

  • Tags look like this: <em>
  • Tags are often paired: <em>football</em>
  • <em> is a start tag
  • </em> is an end tag
  • The <em> tag will emphasise text (often italics)
  • Technically, the em bit of <em> is called an element
  • em is a container tag - it contains text
  • Can also have empty tags, for example, <hr>
  • Empty tags don't have an end tag and don't contain text

Creating a Simple HTML Page

  • Run Windows Notepad
  • Type in some text
  • Save as "example.html" (including quotes, otherwise file will be called example.html.txt) in your N: drive
  • Start a Web browser and open example.html
  • Extra whitespace is ignored and is collapsed to a single space
  • Add <em> tags, save and refresh browser

Simple HTML tags

  • Tags are usually English words or abbreviations
  • <p> - paragraph
  • <br> - new line (an empty tag)
  • <h1> - level 1 heading
  • <h2> - level 2 heading
  • <h3> - level 3 heading
  • <h4> - level 4 heading
  • <h5> - level 5 heading
  • <h6> - level 6 heading

Simple HTML tags

  • Every page should contain:
  • <html></html> - indicating that document contains HTML
  • <head></head> - container for information about the page (e.g. title)
  • <title></title> - container for title of page
  • <body></body> - container for text and other media that make up page

Overall Structure

Overall page structure will be:

<html>
  <head>
    <title>Example Page</title>
  </head>
  <body>
    Page content
  </body>
</html>

Attributes

  • Attributes extend elements by giving them properties
  • Attributes are located after the element in a tag
  • Attributes are often assigned values
  • Values should be surrounded by quotes
  • For example, <p align="center">text</p> (which controls horizontal alignment)
  • Value can also be "left", "right" and "justify"
  • Closing tags don't have attributes
  • So never do <p align="left">text</p align="left">

Possible Attributes

  • Brackets contains details of possible attributes for an element
  • Definitive reference is on W3C site at http://www.w3.org/TR/html4/index/attributes.html

Case

  • Elements and attributes are both case insensitive (but recommend using lower case)
  • Values are case sensitive (align="center" not align="CENTER" or align="Center")
  • Quotes around values can be optional, but recommend that you always use them

Simple Formatting

  • Physical styles:
    • <b> - bold
    • <i> - italics
    • <tt> - teletype (non-proportional font)
    • <u> - underlined text
  • Logical styles:
    • <em> - emphasis
    • <strong> - strong emphasis
    • <code> - computer code (non-proportional font)
    • <cite> - title of a creative work

Lists

  • Two types of list:
    • <ul> - unordered list
    • <ol> - ordered list
  • <li> - list item
  • Lists can be nested
  • Ordered list attributes - type (e.g. type="i") and start (e.g. start="3")

Miscellaneous Tags

  • <blockquote> - long quotation
  • <address> - address
  • <sub> - subscript
  • <sup> - superscript
  • <hr> - horizontal rule
  • <pre> - preformatted text
  • <!-- comments -->

Special Characters

  • <, > and " all appear in HTML tags
  • If we want to display them then we must use these HTML entities:
    • &lt; for a <
    • &gt; for a >
    • &quot; for a "
  • This also means that the & is a special character
  • To display an &, we use &amp;

Hyperlinks

  • To link to another Web page we use an anchor tag (<a>) and an href attribute:
    <a href="URL">Hyperlinked text</a>
  • Link to a document on another Web server (must include http://, the protocol, in the URL):
    <a href="http://www.yahoo.co.uk/">Yahoo!</a>
  • Link to a document in the same directory:
    <a href="otherdocument.html">Other</a>
  • Filenames are case sensitive on a UNIX server (but not on Windows-based Web servers)
  • Recommend that lower case filenames are always used
  • Filenames should not contain spaces

Relative Paths

  • To link to a page in the directory above:
    <a href="../above.html">Up a directory and into a page</a>
  • .. means "go up a directory"
  • To go up two directories:
    <a href="../../furtherabove.html">Up 2 directories</a>
  • To go into a subdirectory:
    <a href="chapter/page.html">Go into subdirectory</a>
  • To link to another directory at the same level:
    <a href="../parallel/doc.html">Parallel document</a>

Absolute Paths

  • Relative paths describe how to find another page relative to the current page
  • Absolute paths give the precise location of a page
  • Absolute paths must start with a /
  • <a href="/iansealy/">Absolute path to a directory</a>

Links Within A Page

  • Mark a point to be linked to using something like:
    <h2 id="bitoftext">My Heading</h2>
  • Link to it using: <a href="#bitoftext">Hyperlink</a>
  • Can also link to that specific point from another document using: <a href="doc.html#bitoftext">Hyperlink</a>

Inline Images

  • Include an image in your page using:
    <img src="image.gif" alt="Image description" width="100" height="100">
  • Must always include alt text
  • Some people can't see images
  • If there is no suitable alt text (e.g. the image is just a spacer) then use alt=""
  • Width and height attributes allow page to be rendered before images have downloaded
