Common Errors

  • Use lower case filenames (task1.html rather than Task1.html) and directory names.

  • Don't include spaces in filenames (my-career.html rather than my career.html) and directory names.

  • Use lower case element and attribute names (not upper case or mixed case).

  • ", &, < and > are special characters and should be encoded as &quot;, &amp;, &lt; and &gt; (even in URLs). Note that they all end with a semi-colon.

  • Text can't be a paragraph and a heading simultaneously:

    <h2 align="center">Heading</h2>
    rather than
    <h2><p align="center">Heading</p></h2>

  • The alt attribute is mandatory for images:

    <img src="image.gif" alt="Image">
    rather than
    <img src="image.gif">

  • Put spaces between attributes:

    <img src="image.gif" alt="Image">
    rather than
    <img src="image.gif"alt="Image">

  • Quote attribute values:

    <p align="center">
    rather than
    <p align=center>

  • </head> is always followed by <body>.

  • Only <title> can go in <head> section (so far).

  • Always close list items with a </li> and paragraphs with a </p>.

  • Nest tags properly:

    <b><i>text</b></i>
    rather than
    <b><i>text</i></b>

  • Where possible, use logical styles rather than physical styles.

  • The first heading of a page will usually be <h1>. Subsections will start with <h2>. Any subsections of those will start with <h3>, etc...

  • Nested lists need to be entirely nested within a list item of the parent list, not between list items. Nothing can go between list items.

  • <h1><strong>Heading</strong></h1> is not necessary as headings are bold by default and already strongly emphasised.

  • Use appropriate tags. A list of things should be marked up as a list.
