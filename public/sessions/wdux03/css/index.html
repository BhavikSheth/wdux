Workaround Problems

Problems with pages that use workarounds:

  • Pages are larger (especially if images are used) and take longer to download
  • Pages are harder to maintain because:
    • the content is more complex
    • the content is harder to read
    • changing the style of a set of pages involves altering every page and often each individual tag (e.g. if want to change <font color="green"> to <font color="red">)

Both of these mean users waste time and so do authors. Much better to separate the content and the styling. Authors can concentrate on the content and designers can easily change the design.

Cascading Stylesheets

Cascading Stylesheets (CSS or just stylesheets) allow content to be separated from formatting

Examples of sites that use CSS page formatting are myriad, but:

  • http://www.espn.com/
  • http://www.theguardian.com/
  • http://www.csszengarden.com/

Simple Example

<style type="text/css">
body { color: red; }
</style>

CSS is the bit between the <style> tags

<style> tag must go in head of page (like the <title> tag).

All text content (everything within the body tags) is red.

Syntax

body { color: red; }

Part before the curly brackets (in this case the body tag) is the selector. Part within the curly brackets is the declaration.

Each declaration consists of a property name (in this case color) and its value (in this case red). The name and value must be separated with a colon.

Can have more than one selector, for example:
h1, h2 { color: red; }

More Syntax

Can have more than one property, for example:
body { color: red; background-color: green; }

Properties must be separated with semi-colons.

A stylesheet is just a collection of these rules.

Location of Styles

  • Internal stylesheet
    <style> tag appears in head of document.
  • External stylesheet
    A .css file contains the styles. Use this with:
    <link rel="stylesheet" type="text/css" href="style.css">
    Goes in the head of document.
    File just contains CSS (no HTML, so no <style> tag).
  • Inline styles
    Within a single HTML tag.
    <p style="color: blue;">

Classes

Styles can be applied to any HTML tags, but what if you want some h1 tags to have one style, but other h1 tags to have another style?

Use classes. Can define different classes of style. For example:

h1.warning { color: red; }
h1.ok { color: green; }

Apply specific class to h1 tags:

<h1 class="warning">Red / Warning Heading</h1>
<h1 class="ok">Green / OK Heading</h1>

More Classes

Don't have to associate classes with a specific tag, for example:

h1.yuk { color: red; background-color: green; }

<h1 class="yuk">Horrid style</h1>
<p class="yuk">Normal style</p>

.yuk { color: red; background-color: green; }

<h1 class="yuk">Horrid style</h1>
<p class="yuk">Horrid style</p>

IDs

IDs are similar to classes, except the syntax is different (a hash rather than a full stop).

#warning { color: red; }
#ok { color: green; }

Apply specific id to h1 tags:

<h1 id="warning">Red / Warning Heading</h1>
<h1 id="ok">Green / OK Heading</h1>

More IDs

Each id must be unique on a page, whereas the same class can appear multiple times.

Can never have:
<p id="intro">First Paragraph</p>
<p id="intro">Second Paragraph</p>

But can have:
<p class="intro">First Paragraph</p>
<p class="intro">Second Paragraph</p>

IDs are widely used in CSS positioning as they're used to specifically identify an area of the page.

Comments

Can add comments to stylesheets using /* and */. For example:

h1 { color: red; } /* Make headings red */
p { color: blue; } /* Make paragraphs blue */

Stylesheet Precedence

What if an inline style says that paragraphs should be blue, but an external stylesheet says they should be green?

Styles have an order of precedence:

  • Inline styles
  • Embedded styles
  • Linked styles
  • Default browser styles

So an inline style overrides an external style.

Font Properties

  • Named fonts
    p { font-family: Verdana, Helvetica, sans-serif; } /* Go from specific to generic font */
  • Font size
    p { font-size: 16pt; }
    Can define size using points (e.g. 16pt), percent (e.g. 150% or 1.5 times larger than the default), ems (e.g. 1.5em or 1.5 times larger than the current default), pixels (e.g. 30px), etc...
    Recommend percent or ems and not points or pixels
  • Font style
    p { font-style: italic; }

More Font Properties

  • Font weight
    p { font-weight: bold; }
    p { font-weight: normal; }
  • Text transform
    p { text-transform: uppercase; }
    p { text-transform: lowercase; }
    p { text-transform: capitalize; }
  • Text decoration
    p { text-decoration: underline; }
    p { text-decoration: overline; }
    p { text-decoration: none; }
    p { text-decoration: line-through; }

div And span Tags

Styles should be applied to the most appropriate tag. For example, if you want to apply a style to a quotation then use the blockquote tag.

blockquote { font-weight: bold; }

<blockquote>Quotation</blockquote>

If you want to apply a style only to specific quotations then use a class.

blockquote.director { font-weight: bold; }

<blockquote class="director">Quotation</blockquote>

More div And span Tags

If there's no suitable tag then use a div or span tag.

A div is equivalent to a block level element (one that normally starts on a new line) like h1 or p, whereas a span is equivalent to an inline element (one that doesn't start a new line) like strong or b.

span.eventlocation { font-weight: bold; }

<span class="eventlocation">Birkbeck College, Malet St</span>

More div And span Tags

div (and span) can be used to group elements rather than repeating classes.

<p class="introduction">First Paragraph</p>
<p class="introduction">Second Paragraph</p>

could instead be:

<div class="introduction">
<p>First Paragraph</p>
<p>Second Paragraph</p>
</div>

Descendant Selectors

Descendant selectors are a great way to avoid repetition.

li.nav { font-weight: bold; }

<ul>
<li class="nav">Home</li>
<li class="nav">About Us</li>
</ul>

could instead be:

ul.nav li { font-weight: bold; }

<ul class="nav">
<li>Home</li>
<li>About Us</li>
</ul>

CSS Support

Modern Web browsers (IE7+, Opera 7+, Firefox, Safari, Google Chrome) all have good CSS support.

Sites that use CSS should always be thoroughly tested on a range of browsers (i.e. different types of browser and different versions of each type of browser).
