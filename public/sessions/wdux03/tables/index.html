Introduction

Tables are used for tabular data

Team Points
Man City 25
Man Utd 20
Tottenham 20

Table Tags

Every table has at least three or four tags:

  • <table> - defines table (one per table)
  • <tr> - defines a table row (one per row)
  • <th> - defines a header cell
  • <td> - defines a data cell

Simple Example

<table>
    <tr>
        <th>Team</th>
        <th>Points</th>
    </tr>
    <tr>
        <td>Man City</td>
        <td>25</td>
    </tr>
    <tr>
        <td>Man Utd</td>
        <td>20</td>
    </tr>
    <tr>
        <td>Tottenham</td>
        <td>20</td>
    </tr>
</table>

Table Attributes

  • border (in pixels)
  • cellspacing (in pixels)
  • cellpadding (in pixels)
  • width (in pixels or percent)
  • align ("left", "right" or "center")

Table Row Attributes

  • align ("left", "right" or "center")
  • valign ("top", "middle", "bottom")
  • bgcolor (background colour of row)

Table Cell Attributes

  • align ("left", "right" or "center")
  • valign ("top", "middle" or "bottom")
  • width (in pixels or percent)
  • bgcolor (background colour of cell)
  • colspan (merge columns)
    cell cell
    colspan="2"
  • rowspan (merge rows)
    cell rowspan="2"
    cell

Nested Tables

  • Can have a whole table inside a single cell of another table
    top left top right
    bottom left
    top left top right
    bottom left bottom right
  • Can nest tables to many levels

Using Tables To Control Layout

  • Technically, tables should only be used to present tabular data (e.g. a profit and loss account)
  • HTML was not designed for layout
  • Layout is best done with CSS, but tables used to be used and often still are for HTML emails
